# VPN连接问题诊断指南

## 📱 如何获取详细日志

### 方法1：使用ADB（推荐）

如果你已经安装了Android SDK，可以使用以下命令查看实时日志：

```bash
# 连接手机后执行
adb logcat | findstr "SusuVpnService"
```

或者查看所有相关日志：
```bash
adb logcat | findstr -i "vpn v2ray susu"
```

### 方法2：查看应用内错误提示

1. 打开速连VPN应用
2. 点击连接按钮
3. 如果出现错误，会显示红色错误卡片
4. 点击错误卡片上的 **"查看诊断信息"** 按钮
5. 截图保存错误信息

## ❓ 常见问题排查

### 问题1：v2ray核心文件缺失

**症状**：点击连接后立即显示"v2ray核心文件不存在"

**原因**：APK构建时v2ray没有正确打包

**解决方法**：
- 确认从最新的GitHub Actions构建下载APK
- 检查构建日志中是否有"✓ arm64-v8a v2ray 已安装"

### 问题2：VPN权限未授予

**症状**：点击连接后没有任何反应，或显示"TUN接口建立失败"

**解决方法**：
- 首次使用时会弹出系统VPN权限对话框
- 必须点击"允许"或"确定"
- 如果错过了，可以在系统设置中重新授权

### 问题3：节点配置错误

**症状**：显示"连接超时"或"节点不可用"

**解决方法**：
- 检查订阅地址是否正确
- 点击首页右上角的刷新按钮更新节点列表
- 尝试切换其他节点

### 问题4：网络问题

**症状**：显示"连接超时（30秒）"

**解决方法**：
- 检查手机网络连接是否正常
- 尝试切换WiFi/移动数据
- 确认节点服务器是否在线

## 🛠️ 调试步骤

### 步骤1：检查订阅是否正常

1. 打开应用
2. 进入"服务器"标签页
3. 点击右上角刷新按钮
4. 观察是否能成功加载节点列表
5. 如果失败，检查订阅地址是否正确

### 步骤2：检查节点信息

1. 在服务器列表中选择一个节点
2. 确认节点有完整的信息（名称、地址、端口）
3. 尝试选择不同类型的节点（如VMess、Shadowsocks等）

### 步骤3：观察连接过程

1. 回到首页
2. 确保已选择一个节点
3. 点击大大的连接按钮
4. 观察状态变化：
   - 应该显示"连接中..."
   - 然后变为"已连接"或显示错误

### 步骤4：查看详细错误

如果连接失败：
1. 首页会显示红色错误卡片
2. 点击"查看诊断信息"按钮
3. 阅读诊断建议
4. 截图保存错误信息

## 📋 需要提供的信息

如果问题仍然存在，请提供以下信息：

1. **错误截图** - 包含完整的错误提示信息
2. **连接日志** - 使用ADB获取的logcat日志
3. **节点类型** - 你尝试连接的节点类型（VMess/VLESS/Shadowsocks等）
4. **Android版本** - 手机的Android系统版本
5. **设备型号** - 手机品牌和型号

## 💡 快速测试

你可以尝试以下步骤快速测试：

```
1. 卸载当前安装的APK
2. 重新从GitHub Actions下载最新APK
3. 安装新APK
4. 打开应用，等待订阅加载完成
5. 选择一个节点
6. 点击连接
7. 观察是否弹出VPN权限对话框
8. 点击"允许"
9. 等待连接结果
```
